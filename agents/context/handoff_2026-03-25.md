# Handoff 2026-03-25

## Big picture: why this work exists

This repository is shifting from ad-hoc simulation experiments to a durable, teaching-first analog design harness.

The long-term goal is not only to publish good labs, but to make the *system* that produces labs improve itself over time. We are building a closed loop:

1. Produce labs (author -> compile -> simulate -> normalize -> analyze -> explain).
2. Capture high-fidelity execution telemetry from raw sessions.
3. Distill recurring friction, failures, and user corrections into structured signals.
4. Promote evidence-backed improvements into prompts, instructions, scripts, and policies.
5. Re-run labs under updated harness rules to verify real improvement.

In short: labs are the workload, telemetry is the signal, and harness updates are the product that compounds.

## Strategic intent for current developments

- Move to ngspice + HDF5-first artifacts to reduce backend variability and simplify downstream tooling.
- Keep artifacts local to each lab to make labs portable and self-contained.
- Standardize ingestion so every session can be mined consistently without ad-hoc CLI usage.
- Improve event heuristics so we detect actionable friction (for example, overly long chained commands) and avoid overreacting to low-value noise.
- Build persistent memory so future executor sessions can fetch proven tactics instead of rediscovering them.
