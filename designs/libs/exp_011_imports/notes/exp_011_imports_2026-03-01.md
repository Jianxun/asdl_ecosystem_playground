## Result
- pass/fail: pass
- key measurements:
  - Compiler resolved imports across all roots in one testbench:
    - `gf180mcu.asdl` from `pdks/gf180mcu/asdl`
    - `analoglib.asdl` and `simulation.xyce.asdl` from `libs_common/`
    - `./ota_5t.asdl` from local `libs/exp_011_imports/`
  - Xyce simulation completed with 1 successful DC step and 0 failed steps.
  - OP formatter produced a pivot table with 6 devices and columns: `CGD, CGS, GDS, GM, VBS, VDS, VDSAT, VGS, VTH`.
- anomalies:
  - First simulation attempt failed due to unbraced symbolic source values (`VDD`, `VCM`) in source parameters.
  - Replacing with `modules.tb.variables` + brace substitution (`{VDD}`, `{VCM}`) resolved to numeric literals at emission time and simulation passed.
  - `save_op` needed `DC` in its template; `.PRINT FORMAT=CSV N(*)` caused an analysis/print mismatch in Xyce.
  - `save_op` CSV output could not coexist with `save` RAW output in the same run because Xyce reported mixed print formats to one file.
  - Import path style is inconsistent across repository examples (`./`, `../`, and root-relative from `libs/`), which makes first-pass authoring error-prone for agents.

## Tooling Notes
- Diagnostics quality observations:
  - Missing file errors are explicit (`PARSE-004: ASDL file not found`) and useful for TDD-style compile-first checks.
- Workflow friction points:
  - Agents must infer whether a non-PDK/non-common import should be written as `./local.asdl`, `../peer/path.asdl`, or a `libs`-root relative path.
  - For portable experiment handoff, external experiment-library dependencies should be copied into the task folder instead of linked across `libs/` subtrees.
- Suggested fixes:
  - Adopt one import-style rule: use bare filenames for `pdks`/`libs_common`, and `./` for task-local experiment files.
