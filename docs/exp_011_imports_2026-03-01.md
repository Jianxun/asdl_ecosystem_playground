## Tooling Notes
- Diagnostics quality observations:
  - Missing file errors are explicit (`PARSE-004: ASDL file not found`) and useful for TDD-style compile-first checks.
- Workflow friction points:
  - Agents must infer whether a non-PDK/non-common import should be written as `./local.asdl`, `../peer/path.asdl`, or a `libs`-root relative path.
  - Mixing nested relative imports (`../`) with `libs`-root imports increases cognitive load during refactors.
- Suggested fixes:
  - Adopt one import-style rule: use bare filenames for `pdks`/`libs_common`, `./` for local siblings, and `libs`-root-relative paths for cross-library imports.
